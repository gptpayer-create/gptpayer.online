Frequently Asked Questions

Can this convert scanned PDFs?
Scanned PDFs are image-based, so text extraction is limited without OCR. This tool uses PDF.js text extraction which works best on PDFs created digitally (e.g., exported from Word or Google Docs). For scanned PDFs, OCR software like Adobe Acrobat is recommended.
Will tables and images be included?
Images and complex graphic layouts are not included in the DOCX output — only text content is extracted. Tables may appear as plain text depending on how they were created in the PDF.
Is my PDF uploaded to a server?
No. All text extraction happens in your browser using PDF.js. The file never leaves your device. The DOCX is also generated locally using the docx.js library.
What is the DOCX file compatible with?
The output is a standard .docx file compatible with Microsoft Word (2007+), Google Docs, LibreOffice Writer, and Apple Pages.
Can I extract only specific pages?
Currently the tool extracts all pages. You can delete unwanted page sections in Word after downloading. A page range selector is on our roadmap for a future update.

About PDF to Word Conversion

Converting a PDF back to an editable Word document lets you modify content that was previously locked in PDF format. This is useful when you need to edit a report, update a contract, extract text for reuse, or make corrections to a finalized document.

How PDF to Word Conversion Works

Our tool uses PDF.js to extract all text content from each page of your PDF and organizes it into a structured DOCX file using the docx.js library. The process preserves the page structure — each PDF page becomes a clearly labeled section in the Word document. This makes it easy to navigate and edit specific parts of long documents.

Important Limitation — Scanned PDFs

There are two types of PDFs: digital PDFs (created from Word, Google Docs, or other software) and image-based PDFs (scanned physical documents). Our tool extracts text from digital PDFs reliably. Scanned PDFs are essentially photographs of pages — there is no selectable text embedded in them. For scanned PDFs, OCR (Optical Character Recognition) software like Adobe Acrobat or Google Drive's built-in OCR is needed to recognize and extract the text.

What Is and Is Not Preserved

Text content, paragraph structure, and page organization are preserved in the output DOCX. Complex visual formatting (multi-column layouts, custom fonts, decorative elements), images, tables with complex borders, and vector graphics are not included — only the text content is extracted. For simple text-heavy PDFs like reports and articles, the output is clean and immediately usable.

Customizing the Output

Before generating the DOCX, you can choose the font family (Times New Roman, Calibri, Arial, or Georgia) and font size (10pt to 14pt) of the output document. You can also choose whether to include page labels — section markers indicating which page of the original PDF each section corresponds to. This makes it easier to cross-reference the output with the original.

Frequently Asked Questions

Why can't I extract text from my PDF? If the PDF is scanned (image-based), text extraction will not work without OCR. Try opening the PDF and attempting to select text with your cursor — if you cannot select text, it is an image-based PDF.

Will tables be extracted correctly? Simple tables may appear as plain text. Complex tables with merged cells may not preserve their structure in the DOCX output.

Is my PDF uploaded to a server? No. All processing is done locally in your browser using PDF.js and docx.js. Your file never leaves your device.
