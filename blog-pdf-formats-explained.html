PDF/A, PDF/X, PDF/E — The Different PDF Standards Explained

Published: April 3, 2026  ·  10 min read  ·  GPTPayer Team

Most people think of PDF as a single, universal format — open a PDF, read the document. But PDF actually has a family of specialized variants, each designed for a specific professional context. If you have ever been told to submit a "PDF/A compliant document" by a government agency, or received a print shop request for "PDF/X-4 format," this guide explains exactly what those mean and when each variant is required.

Standard PDF — The Everyday Format

The standard PDF format (now at version PDF 2.0 as of ISO 32000-2) is what most of us use every day. It supports interactive elements such as fillable forms, hyperlinks, embedded audio and video, digital signatures, and JavaScript. It can be encrypted with passwords, annotated, and contains no restrictions on content type.

Standard PDF is the right format for personal documents, business communications, invoices, reports, contracts, and any document where you want maximum feature support. This is what GPTPayer.online and most other PDF tools create by default.

PDF/A — Built for Long-Term Archiving

The "A" stands for Archive. PDF/A is an ISO standard (ISO 19005) specifically designed to ensure that a document will still be readable and look identical in 50 or even 100 years, regardless of what software exists at that time. The core principle is self-containment — a PDF/A file must contain everything needed to render it, with no reliance on external resources.

To achieve this, PDF/A has strict requirements that differ from standard PDF:

  • All fonts must be embedded within the file — no reliance on system fonts that may not exist in the future
  • Color spaces must be fully specified — so colors render identically on any device
  • Encryption is not permitted — an encrypted document cannot be guaranteed to be openable in the future
  • External content is not allowed — no hyperlinks to external websites, no embedded videos, no audio
  • JavaScript is prohibited — no executable code
  • Transparency is managed carefully — depending on the conformance level, transparency may need to be flattened

Government agencies, law firms, banks, universities, and medical institutions use PDF/A to ensure that official records remain permanently accessible and visually unchanged. In many countries, PDF/A is legally mandated for court filings, tax records, and official submissions.

PDF/A Conformance Levels

PDF/A comes in several conformance levels. PDF/A-1 is the original, strictest standard. PDF/A-2 and PDF/A-3 add support for JPEG 2000 compression, transparent objects, and embedded attachments (PDF/A-3 allows any file type as an attachment, not just PDF). For most archiving purposes, PDF/A-2b (basic conformance, PDF/A-2) is the recommended choice.

PDF/X — Designed for the Printing Industry

The "X" stands for Exchange. PDF/X was developed for the graphic arts and commercial printing industry to solve a specific problem: designers creating files on their computers that look correct on screen, but print with wrong colors or missing fonts when sent to a professional printing press.

PDF/X enforces standards that guarantee print fidelity:

  • CMYK color spaces: Print uses Cyan, Magenta, Yellow, Black inks — not RGB (Red, Green, Blue) used by screens. PDF/X ensures all colors are specified in CMYK.
  • Embedded fonts: All fonts must be embedded so the print shop uses exactly the typeface the designer intended.
  • Bleed and trim marks: PDF/X requires that bleed areas (the extra content beyond the cut edge) and trim box (the final cut size) are properly defined.
  • No transparency: In PDF/X-1a, all transparency must be flattened to avoid rendering differences across RIP processors. PDF/X-4 relaxes this restriction.

If you are submitting files to a printing press, magazine publisher, or book printer, they will almost certainly request PDF/X-1a (strict, CMYK only) or PDF/X-4 (supports transparency and layers). Always check with your print provider for their specific requirements.

PDF/E — Developed for Engineering

The "E" stands for Engineering. PDF/E was developed for the engineering, manufacturing, and geospatial industries where documents often include interactive 3D content, CAD drawings, and complex technical diagrams.

PDF/E supports 3D models, interactive engineering drawings, and U3D (Universal 3D) content that can be rotated and examined within the PDF viewer. It is designed for technical exchange between manufacturers, suppliers, and regulatory bodies where the exact visual representation of engineering data must be preserved.

Unless you work in manufacturing, aerospace, architecture, or similar technical fields, you are unlikely to encounter PDF/E in your daily work.

PDF/UA — Universal Accessibility

PDF/UA (Universal Accessibility) is an ISO standard (ISO 14289) designed to ensure PDF documents are accessible to people with disabilities — particularly those using screen readers or other assistive technologies. A PDF/UA compliant document has properly tagged content (headings, paragraphs, lists, tables are all tagged with semantic meaning), alt text for images, a logical reading order, and accessible form fields.

PDF/UA is increasingly required for government documents, educational materials, and corporate publications in many countries under accessibility legislation such as the ADA (Americans with Disabilities Act) and the EU Web Accessibility Directive.

How to Create PDF/A Documents

Most standard PDF creation tools do not produce PDF/A by default. To create a PDF/A document:

  • Microsoft Word: File → Save As → PDF → Options → check "ISO 19005-1 compliant (PDF/A)"
  • LibreOffice: File → Export as PDF → General tab → check "PDF/A-1a"
  • Adobe Acrobat Pro: File → Save As → PDF/A, then choose the conformance level
  • Online converters: Several services like PDF24 and Smallpdf offer PDF/A conversion

Frequently Asked Questions

How can I tell which PDF variant a file is?

In Adobe Acrobat Reader, go to File → Properties → Description tab. The PDF Version and any conformance standard (PDF/A, PDF/X) will be listed there. Alternatively, the file may contain XMP metadata specifying the conformance level.

Can I convert a regular PDF to PDF/A?

Yes, but it requires processing. Adobe Acrobat Pro has a built-in PDF/A conversion tool. LibreOffice can also convert to PDF/A by opening and re-exporting. Note that PDFs with embedded videos, JavaScript, or encryption cannot be converted to PDF/A without removing those elements.

Does a government portal that requires PDF accept standard PDF or only PDF/A?

Most government portals accept standard PDF. PDF/A is specifically required when the government wants to archive the document permanently — this is common for tax submissions, court filings, and official records in some jurisdictions. The portal's requirements page will specify if PDF/A is needed.
